JavaScript Objects vs Ruby Hashes

Sunday Feb 22, 2015

This week of Phase 0 at DBC had us learning a little JavaScript. After writing in Ruby for the past couple months, it was surprisingly easy to transition to JavaScript for me. Once you get past the syntax differences, the two languages seem to me to have similar logic. One such example that jumped out in front of me was Ruby hashes compared to JavaScript objects. This is what I’ll choose to examine more closely for this week’s blog.

Let’s go with the new language in my life and start with JavaScript (JS) and its objects. Declaring a JS object is quite similar to declaring a Ruby hash; both can us the literal notation. In JS, you’d write: 

 
var jsObject = {
  key1: value2,
  key2: value2
}

If you want to access, say, value 2, you;d write jsObject.key2. This would return value2. You could even add another key/value pair by writing: 


  jsObject.key3 =  “value3”

This looks nearly identical to Ruby hashes. In terms of functionality, there really do not seem to be a big difference here. Ruby follows similar syntax for creating a hash. Write it the exact same way as above, but drop the “var” and you have a Ruby hash.
