Ruby Enumerables

Enumerable#cycle

Tuesday Feb 3, 2015

After doing a lot of digging through Google, I learned about the Enumerable#cycle method and it’s pretty [bleeping] cool. To start, I’ll walk you through the basic stuff and once we get that down, I’ll show you how the fun tricks with it.
At its core, the #cycle method runs a block either n times or, if you don’t specify, forever. You set it up like this: 

      
      >> shopping_list = ["apples", "bananas", "grapes"]
      => ["apples", "bananas", "grapes"]
      >> shopping_list.cycle {|x| puts x}
      apples
      bananas
      grapes
      apples
      bananas
      grapes
      apples
      bananas
      grapes
      apples
      bananas
      grapes
      [... and on and on and on. This is where I hit control + c]

Pretty simple. Now lets add our number to break it after that many cycles. 

    
   >> shopping_list. cycle(2) {|x| puts x}
  apples
  bananas
  grapes
  apples
  bananas
  grapes
  => nil

Now, to get a little fancier, you can write the break inside of the block: 


>> shopping_list.cycle do |x|
>> puts x
>> break if x == "grapes"
>> end
apples
bananas
grapes
=> nil

Not amused? Well maybe this will get ya going: If you set a variable equal to the result of running #cycle on an Enumerable, then that object can have #next called on it to return the next item in the sequence. Let me clarify with an example: 


>> food_items = shopping_list.cycle
=> #
>> food_items.next
=> "apples"
>> food_items.next
=> "bananas"
>> food_items.next
=> "grapes"
>> food_items.next
=> "apples"

Is that cool enough for you?! You can even call #rewind on the object and have it start again from the beginning of the cycle! Watch it in action (the following code continues from where the previous code left off) : 


>> food_items.next
=> "apples"
>> food_items.next
=> "bananas"
>> food_items.rewind
=> #
>> food_items.next
=> "apples"
