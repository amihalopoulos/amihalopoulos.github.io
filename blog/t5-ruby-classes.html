Classes

Friday Jan 16, 2015

So you probably know how to declare variables and even define your own methods. Well, now it is time to start storing some of that knowledge into your very own classes! Why would you want to make your own class, you ask? Well say you want to create a legion of minions in your program. Each one has the abilities to cook, sing, clean, make jokes, etc. Instead of writing new methods every single time you want to create a minion, you could write a class called minion. Each instance of this class will be represent each invidivual minion. Below I create a class Minion and make three new instances. 

class Minion
  def initialize (name, eyes)
  @name = name
  @eyes = eyes
  end
end

kevin = Minion.new("Kevin", 2)
steve = Minion.new("Steve", 1)

One the first line, I named the new class Minion. Then I defined the initialize method. This method is very useful because it automatically runs everything inside it as soon as an instance of the class is created. The method takes two arguments, name and number of eyes. It then assigns each of those arguments to an instance varaible. You assign instance variables with the @ sign. These variables will be available for every subsequent method in its individual instance of the class Minion and only that instance. For example, the variable @eyes for kevin will not affect Steve ever, in any way. The two are completely separate. On the last two lines I made an instance of Minion named kevin and one of steve.
